## What Is TANDRmgr?

**TANDRmgr** (TANDR Manager) is the **API Orchestrator Server** - a central hub that:

1. **Routes requests** to multiple LLM providers (Gemini, Anthropic Claude, Ollama)
2. **Relays messages** to different TANDR services (agents, bots, social)
3. **Manages authentication** and security for all service calls
4. **Provides a chat interface** at the root (http://127.0.0.1:8085/)

### Technology
- **Framework:** Node.js + Express.js (server.js on port 8085)
- **Request handling:** PHP CGI executor for backend scripts
- **API format:** REST endpoints at `/api/:script`
- **Logging:** Append-only logs in `/adir/logs/`

## API Endpoints

TANDRmgr exposes endpoints at `/api/:script` that execute PHP scripts.

### Discovered Endpoints

1. **`/api/mgr.php`** - Manager/Status Endpoint
   - **Action:** `status` (verified working)
   - **Response:** Full configuration dump with all services, LLM providers, enabled status
   - **Usage:** Primary way to check TANDRmgr health and service configuration
   - **Test:** `curl -X POST "http://127.0.0.1:8085/api/mgr.php" -H "Content-Type: application/json" -d '{"action":"status"}'`

2. **`/api/providers/gemini.php`** - Gemini LLM Interface
   - **Purpose:** Routes requests to Google Gemini API
   - **Model:** gemini-2.0-flash-exp
   - **Status:** Active

3. **`/api/providers/anthropic.php`** - Anthropic Claude Interface
   - **Purpose:** Routes requests to Claude API (help mode)
   - **Model:** claude-opus-4-6
   - **Status:** Available

4. **`/api/providers/ollama.php`** - Local Ollama Interface
   - **Purpose:** Routes requests to local Ollama instance
   - **Port:** 11434
   - **Status:** Available

5. **`/api/security.php`** - Security/Authentication Handler
   - **Purpose:** Manages authentication and security policies
   - **Status:** Active

### Relay Mechanism

TANDRmgr doesn't have individual endpoints for each service. Instead, it:
1. Receives a message at the chat interface (index.html)
2. Routes it based on configuration to the appropriate service
3. The service endpoint (jerry's 9200, bot 8081, etc.) handles the request
4. TANDRmgr receives the response and returns it to the chat interface
