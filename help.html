Getting Started

What is FOB?

FOB is a local AI agent system that runs on your computer. It launches several small servers that each do a specific job — chat agents, knowledge bots, and tools for building more of them. Everything runs locally. Your data stays on your machine.

Starting Up

  1. Double-click the V3AM FOB desktop shortcut, or launch C:\FOB\V3AMFOB\V3AMFOB.exe directly.
  2. V3AMFOB starts all services automatically in the background. The rack panel shows a status card for each service — watch for the indicators to go green.
  3. Click any service card to open it in your browser.
Tip: If a card stays red, click the restart button on that card or check the log panel for error output.

Shutting Down

  1. To keep services running: Click the X on V3AMFOB — it minimizes to the system tray. Services keep running in the background.
  2. To stop everything: Right-click the tray icon → Quit. This kills all services and exits cleanly.

You can also run C:\FOB\STOP-ALL2.bat to kill all services from the command line.

Important: Clicking X does NOT stop the services — it just hides the window. Use right-click → Quit, or STOP-ALL2.bat to actually shut down.

Setting Up API Keys

Agents need an AI model to work. You have two options:

Option A: Local models (free, private) — Install Ollama, then pull a model:

  1. Install Ollama from ollama.com
  2. Open a terminal and run: ollama pull qwen2.5:7b
  3. Ollama runs automatically on port 11434. Agents will find it.

Option B: Cloud models (faster, smarter) — Get an API key from one of these providers:

Google Gemini (recommended)

Where to enter your key:

  1. Open the agent or bot you want to configure (click its card on the Command Center).
  2. Go to the Settings tab in the chat interface.
  3. Paste your API key into the provider field (Gemini, Anthropic, or OpenAI).
  4. Select that provider as active. Click Save. The agent will use it immediately.
Tip: Each agent has its own config. You can run different agents on different models — a cheap local model for simple tasks, a cloud model for complex ones.

Troubleshooting

  1. Agent says "PHP not found" — Make sure C:\FOB\php\php-cgi.exe exists. It should already be bundled.
  2. No response from agent — Check that you have an API key set (Settings tab) or Ollama running. No model = no response.
  3. "Port already in use" — Run C:\FOB\STOP-ALL2.bat first, then relaunch V3AMFOB.
  4. Voice not working — Use Chrome. Click the mic button once (don't double-click). Allow microphone permission when prompted.
  5. Services won't start — Open the rack panel in V3AMFOB to see service status and logs. Click the restart button on the failed card.
