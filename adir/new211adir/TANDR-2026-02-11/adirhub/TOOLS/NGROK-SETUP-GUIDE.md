## Step 2: Get Your Auth Token

### From Your ngrok Dashboard

1. Log into **ngrok.com**
2. Go to: **Your Authtoken** (left menu)
3. Click **"Copy"** to copy your token
4. Save it somewhere safe (you'll use it twice)

**Your token looks like:** `2c7o...s9Q` (long alphanumeric string)

---

## Step 3: Reserve Static Domains (Optional But Recommended)

### Why Reserve Domains?

By default, ngrok gives you a random domain each time (like `abc-123.ngrok.app`). If you want the **same domain every time**, reserve one.

### How to Reserve

1. Log into **ngrok.com**
2. Go to: **Domains** (left menu)
3. Click **"New Domain"** or **"Reserve Domain"**
4. Enter desired domain:
   - For Agent-Dropper: `agent-dropper`
   - For KB-Maker: `kb-maker`
   - For generated bots: `kb-maker-bot-001`, etc.
5. Click **Reserve**
6. You now own: `agent-dropper.ngrok.app`, `kb-maker.ngrok.app`, etc.

**Cost:** Free tier includes 1 reserved domain. Additional domains available in paid plans.

## Step 4: Configure Agent-Dropper v2

### Option A: With Reserved Domain (Recommended)

**File:** `C:\FOB\adir\new211adir\TANDR-2026-02-11\adirhub\TOOLS\Agent-Dropper-v2\config.json`

**Find this section:**
```json
"ngrok": {
    "enabled": true,
    "auth_token": "placeholder_token_here",
    "domain": "agent-dropper.ngrok.app",
    "status": "placeholder",
    "auto_start": false
}
```

**Edit to:**
```json
"ngrok": {
    "enabled": true,
    "auth_token": "YOUR_AUTH_TOKEN_HERE",
    "domain": "agent-dropper.ngrok.app",
    "status": "configured",
    "auto_start": true
}
```

**Steps:**
1. Open config.json in text editor
2. Find `"auth_token": "placeholder_token_here"`
3. Replace with your ngrok auth token
4. Change `"status"` from `"placeholder"` to `"configured"`
5. Change `"auto_start"` from `false` to `true` (optional - auto-starts tunnel on startup)
6. Save file

### Option B: Without Reserved Domain (Random Each Time)

**Steps:**
1. Same as Option A
2. System will use ngrok's random domain each time
3. Check console after startup for URL
4. URL is temporary (~2 hours)

   ```bash
   # Look for missing commas or quotes
   ```

### ngrok Tunnel Stops After 2 Hours

**Why:** Free tier tunnels timeout periodically

**Solution:** Restart ngrok
```bash
# Option 1: Restart tool
# Option 2: Stop and restart ngrok process
taskkill /F /IM ngrok.exe
# Then restart Agent-Dropper/KB-Maker
```

**Upgrade:** Paid ngrok account for persistent tunnels

### Can Access Locally, But Not Remotely

**Problem:** `http://127.0.0.1:9210` works, but `https://agent-dropper.ngrok.app` doesn't

**Solutions:**
1. Verify ngrok tunnel is running
   ```bash
   tasklist | findstr ngrok
   ```

2. Check ngrok status in tool UI
   - Should show "Tunnel Active"
   - Should show "Connected: Yes"

3. Try from different network
   - WiFi vs mobile data
   - Colleague's computer

4. Check firewall
   - Windows Defender may block ngrok
   - Try disabling temporarily to test

### Domain Keeps Changing

**Why:** You're using ngrok free tier (random domain)

**Solution:** Reserve a static domain
1. Go to ngrok.com → Domains
2. Click "Reserve Domain"
3. Get permanent domain
4. Configure tool to use it

## Advanced: Multiple Instances

### Agent-Dropper - Multiple Instances

If you want to run multiple Agent-Dropper instances:

1. **Instance 1:** Port 9210 → agent-dropper.ngrok.app
2. **Instance 2:** Port 9211 → agent-dropper-2.ngrok.app (reserve domain)
3. **Instance 3:** Port 9212 → agent-dropper-3.ngrok.app (reserve domain)

**Per Instance:**
1. Copy Agent-Dropper-v2 folder
2. Rename to Agent-Dropper-v2-Instance2
3. Update config.json:
   - Port: 9211
   - Domain: agent-dropper-2.ngrok.app
   - Token: same
4. Start each independently

### KB-Maker - Generated Bots

Each bot gets auto-assigned:
- Port: 9221, 9222, 9223, etc.
- Domain: kb-maker-bot-001, kb-maker-bot-002, etc.
- Separate config files
- All using same auth token

## Best Practices

### DO:
✅ Keep auth token secure (don't share publicly)
✅ Reserve domains for permanent tools
✅ Monitor active tunnels in ngrok dashboard
✅ Restart tunnels if they drop
✅ Document your domains (which tool uses which domain)

### DON'T:
❌ Commit auth tokens to git/repositories
❌ Share ngrok URLs on public forums
❌ Leave tunnels running 24/7 without monitoring
❌ Use same domain for multiple tools
❌ Update domain without restarting tools
