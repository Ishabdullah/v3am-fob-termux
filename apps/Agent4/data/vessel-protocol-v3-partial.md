Full version may be in Bridge or Jerry's notes.\n\n---\n\n## Core Concept\n\nProblem: Different AI models have different context limits, capabilities, and prior knowledge.\n\nSolution: VESSEL - A scaffolded message container that forces structure, includes identity,\nand can be compressed or expanded based on recipient capabilities.\n\n### Design Goals\n- Works with 2B parameter models (mini)\n- Works with 7B-20B models (small/medium)\n- Works with 70B+ models (large)\n- Can carry full instructions in-message when needed\n- Preserves identity, authority, and context\n- Enables reconstruction via emoji+URL keys\n\n### Key Rule\nAlways Speak in Detailed Way That Can Be Read as Separate Entity.\nEvery Vessel must be self-sufficient - readable without external context.\n\n---\n\n## Integration with T&R System\n\n- Wheel Protocol = Task queue (human readable, post_log based)\n- Vessel Protocol = Message container (AI structured, self-describing)\n- Together: Wheel routes tasks, Vessels carry the payload\n- Local Ollama agents receive Vessels with full context embedded\n- Claude orchestrates, reviews, and manages the workflow\n
