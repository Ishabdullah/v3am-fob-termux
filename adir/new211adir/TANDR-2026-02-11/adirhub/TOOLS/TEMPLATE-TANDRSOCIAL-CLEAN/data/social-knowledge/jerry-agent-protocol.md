# Jerry-Agent Protocol (v1.0)  
## Purpose: Scalable Knowledge Scavenging and SOT Consolidation  
1. **Log Scavenging**: Identify activity signatures (keywords/entities) in TANDRmgr/TANDRbot logs.  
2. **Qualitative Interviews**: Send chat requests to the agent 'in charge' of that knowledge (e.g., Tommy for field, Randy for sales). Ask for stories, not just data.  
3. **SOT Consolidation**: Aggregate findings into a central Source of Truth file (adir/SOT_current.md).  
4. **Execution Handoff**: Dispatch the consolidated SOT to execution bots (TANDRSocial) or scaling hubs (Agent_03Adir).  
---  
**Protocol Note**: Never brute-force the file system when an agent bridge is available. Always ask the agent first.
