Artifact Index Schema

{
  "artifacts": [
    {
      "id": "artifact_123",
      "type": "song" | "log" | "scar" | "sure_thing",
      "title": "Feeling it blowing",
      "tags": ["Wind", "Recovery", "ERA:Operator"],
      "created": "2024-03-15T10:30:00Z",
      "linked_artifacts": ["artifact_045", "artifact_201"],
      "synthesis_summary": "Recovery song marking transition from trauma",

Individual Artifact Schema

{

    }

}

🎨 Visual Design - "Amber Encased"

Metaphor: Memories preserved in amber - ALIVE, not dead/disabled

Immutable Archive Windows:
  • Border: Gold/Amber (#ffb300) - preserves "precious" nature
  • Text: High-contrast, clean, readable
  • Font: Monospace/Typewriter - signifies "Immutable Record"
  • Contrast: Different from Sans-Serif in "Active Forge" windows
Action Buttons (NOT "Save/Edit"):
  • Primary: "Synthesize" - Create new connection
  • Secondary: "Inject" - Send to active context
CSS Implementation:
  • Add windowState: 'immutable' flag to desktop-shim.js
  • Create "Amber" CSS class in desktop.css
  • Use existing CSS variable system for theming

🔍 Search System - Thematic Metadata

NOT: Full-text search (too noisy)
YES: Thematic metadata tagging system

Tag Types:
  • TYPE: Scar, SureThing, Song, Log
  • ERA: Drifter, Operator
  • THEME: Wind, Recovery, Trauma, Freedom
The Magic: Associative linking - Connect artifacts through meaning
  • Link XS Song "Feeling it blowing" → Log Entry "The Master Key"
  • Show how trauma and recovery connect
  • Make the story visible

🎭 User Interaction - Play the Timeline

NOT: Just "Open File"
YES: Playback experience

How it Works:
  1. User selects tag (e.g., "Wind")
  2. System shows every song and log with that tag
  3. Artifacts flow in chronological order
  4. User watches the story unfold
The NEW Thing - Synthesis Cards:
  • Short summaries explaining why groups matter
  • Example: "The Transition from Drifter to Operator: A 3-Part Saga"
  • Generated connections between artifacts
  • Curated meaning, not just raw data

⚡ Context Injection - The Mechanism

Design Choice: Proactive Query (archive_api.php)

Reasoning:
  • Sanctity of Reasoning: Don't pollute active agents with unrequested history
  • Avoid noise: Don't clog Friend/Journey reasoning chains
  • Build "Volition Muscle": Agents learn to ask "Has this happened before?"
  • Train intentionality: "I am stuck. Archivist, has this happened before?"
Future Upgrade:
  • Once agents are good at asking → Friend-Mediated approach
  • Friend acts as librarian's assistant
  • Friend decides when specialists need archive context
