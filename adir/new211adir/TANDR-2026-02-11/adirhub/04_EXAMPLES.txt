  Opens adir/SESSION.txt
  Sees: "Working on Task X, 60% complete, tests half-done"

Step 3: Read CURRENT.txt

Step 4: Read WORKING.txt

  Sees last 3 actions:
    14:15 Started writing tests

  Knows exactly where to pick up
  No lost context
  No re-reading of unrelated files
```

Result: Instant recovery, full context.

## SCENARIO 4: Agent Coordination (Wheel Protocol)

### Two agents working on same project

#### Agent1 finishes a task:
```
Opens: adir/FEED.txt
Adds: [Agent1] Completed: API endpoints. Next: Unit tests. For: Agent2.
```

#### Agent2 arrives:
```
Opens: adir/FEED.txt
Reads: [Agent1] Completed: API endpoints. Next: Unit tests. For: Agent2.
Knows: Task handed off, tests needed
Opens: adir/CURRENT.txt
Adds to IMMEDIATE: "[ ] Write unit tests for API"
Adds to log: [YYYY-MM-DD HH:MM] Agent2 Claimed unit test task
```

#### Agent2 finishes:
```
Opens: adir/CURRENT.txt
Marks tests complete [x]
Opens: adir/FEED.txt
Adds: [Agent2] Completed: Unit tests. Next: Integration tests. For: Agent3.
```

Result: Clear hand-off, no context lost, each agent knows what's next.

## SCENARIO 5: Read-Only Agent (Can Still Navigate)

### Agent has read-only access
```
Can: Read all adir/ files
Can: Understand task structure
Can: Understand project context
Can: Navigate existing paths
Can't: Create new adir/ directories
Can't: Update files
```

### Read-only workflow:
```
Step 1: Navigate to C:\MyDrive\
Step 2: Read adir/CURRENT.txt → Sees all tasks
Step 3: Read adir/WORKING.txt → Understands context
Step 4: Read adir/FEED.txt → Sees messages
Step 5: Read adir/LOGS/action_log.txt → Understands history
Step 6: Reports findings (via external channel)

Result: Can understand entire project state
        Can navigate without building new stops
        Can contribute through documented insights
```

This is useful for:
- Read-only access to analyze projects
- Agents that can research but not change state
- External audits of project status

## 🎯 KEY PATTERNS

### Pattern 1: New Directory
```
1. Check for adir/
2. If not found → Create it
3. Copy 01-04 files
4. Copy 05_TEMPLATES/
5. Create coordination files from templates
6. Start working
```

### Pattern 2: Returning to Known Directory
```
1. Read CURRENT.txt
2. Read WORKING.txt
3. Read SESSION.txt (if exists)
4. Resume work
```

### Pattern 3: Agent Handoff
```
Agent1:
  Update CURRENT.txt
  Post in FEED.txt: [Agent1] Completed: X. Next: Y. For: Agent2.
  Update SESSION.txt

Agent2:
  Read FEED.txt
  See handoff message
  Read SESSION.txt for context
  Continue from where Agent1 left
```

### Pattern 4: Read-Only Access
```
1. Navigate to any directory with adir/
2. Read all coordination files
3. Understand entire project state
4. Can't update, but can report findings
```

---

## 💡 SUCCESS TIPS

✅ Always read files in ARRIVAL order
✅ Update CURRENT.txt every time tasks change
✅ Keep FEED.txt entries short and clear
✅ Log every action with timestamps
✅ Use SESSION.txt for complex projects
✅ Don't skip LOGS/action_log.txt for recovery
